Chapter 02

로그파일과 데이터파일

1. 로그파일의 종류와 작성

로그파일은 Results 창의 모든 명령문과 output을 텍스트 파일 형태로 저장한 파일입니다.

왜 로그파일을 사용해야 할까?

Results 창은 용량 제한(32KB)이 있어 결과물이 많아지면 이전 내용이 사라질 수 있습니다. 로그파일을 사용하면 모든 작업 기록을 영구적으로 보관, 열람, 편집할 수 있습니다.

로그파일 종류

*.smcl 형식

장점: Result 화면과 동일한 형태로 깔끔하게 보임.

단점: Stata에서만 열 수 있음.

*.log 형식

장점: 일반 텍스트 파일로, MS-Word 등에서 편집 가능.

단점: Stata 화면과 서식이 달라 보일 수 있음.

로그파일 지정 및 관리

로그파일 시작

log using "C:\data\log1.smcl"

로그파일 종료

log close

기존 로그파일 이어 사용

log using 명령어에 옵션을 추가합니다.

  • append: 기존 파일에 내용 추가
  • replace: 기존 파일 덮어쓰기

2. STATA 데이터 파일의 형태

Stata 데이터는 행과 열로 구성된 표 형태입니다.

  • 각 열 (column)은 변수 (variable)를 의미합니다.
  • 각 행 (row)은 관측치 (observation)를 나타냅니다.

변수는 값의 종류에 따라 숫자(numeric) 또는 문자(string) 변수로 나뉩니다.

3. 데이터 저장 형태 (Storage Type)

describe 명령어로 변수의 저장 형태를 확인할 수 있습니다.

byte, int, long: 정수형 숫자
float, double: 실수형 숫자
string (e.g., str18): 문자열

저장 형태는 데이터의 크기와 종류에 따라 메모리 사용 효율을 결정합니다. 일반적으로 Stata가 자동으로 최적의 형태를 지정합니다.

4. 데이터 디스플레이 포맷

실제 값은 그대로 두고, 화면에 보이는 숫자/문자의 형식을 변경할 수 있습니다.

  • e_fmt (%9.2e): 지수 형태 (아주 크거나 작은 수 표현에 용이)
  • f_fmt (%9.2f): 고정 소수점 형태 (소수점 이하 자릿수 고정)
  • g_fmt (%8.1g): 일반 형태 (기본값)
  • s_fmt (%-9s): 문자열 형태 (-는 왼쪽 정렬)

포맷 변경 명령어:

format g_fmt %11.2g

5. STATA do-file

do-file은 Stata 명령어들을 순서대로 저장해 놓은 스크립트 파일 (*.do) 입니다.

do-file을 사용하면 복잡한 분석 작업을 저장하고, 반복 실행하며, 다른 사람과 공유하기 용이해져 작업의 효율성과 재현성을 크게 높일 수 있습니다.
