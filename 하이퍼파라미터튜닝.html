하이퍼파라미터
튜닝 가이드

모델의 성능을 극대화하는 방법은 다양합니다.
학습 데이터를 정제하거나 중요한 변수를 선택하는 것도 중요하지만,
오늘은 모델의 하이퍼파라미터를 최적화하는 방법에 대해 깊이 있게 알아봅니다.

개념 바로잡기

하이퍼파라미터 (Hyperparameter)

사용자가 직접 설정하여 모델의 학습 방식을 조절하는 값입니다. 도메인 지식을 기반으로 수동 설정하거나, 튜닝 알고리즘을 통해 찾습니다.

  • 의사결정나무의 최대 깊이 (max_depth)
  • 딥러닝의 학습률 (Learning Rate)
  • 학습 반복 횟수 (Epochs)

파라미터 (Parameter)

데이터 학습 과정에서 자동으로 결정되는 값입니다. 사용자가 직접 설정하는 것이 아니라 모델이 데이터에 맞춰 최적화합니다.

  • 회귀 모델의 계수 (Weight)
  • 절편 (Bias)
  • 신경망 내부의 가중치
Method 01

그리드 서치 (Grid Search)

'Grid'는 격자를 의미합니다. 촘촘한 격자처럼, 우리가 지정한 모든 하이퍼파라미터 후보들의 조합을 시도해보는 방식입니다. 가장 확실하게 최적의 값을 찾을 수 있지만, 후보가 많아질수록 시간이 매우 오래 걸린다는 단점이 있습니다.

정확도 모든 조합 확인
속도 느림 (비용 높음)

탐색 공간 시각화

모든 교차점을 균일하게 탐색

grid_search_example.py 
from sklearn.model_selection import GridSearchCV        # 그리드 서치 모듈
from sklearn.ensemble import RandomForestClassifier     # 모델 로드

# 1. 모델 생성
model = RandomForestClassifier()

# 2. 탐색할 하이퍼파라미터 그리드(딕셔너리) 설정
param_grid = {
    'n_estimators': [10, 50, 100, 200],      # 결정 트리(Tree)의 개수
    'max_features': ['auto', 'sqrt', 'log2'], # 분할 시 고려할 특성의 수
    'max_depth': [4, 5, 6, 7, 8],            # 트리의 최대 깊이 (과적합 방지)
    'criterion': ['gini', 'entropy']         # 불순도 측정 지표
}

# 3. GridSearchCV 객체 생성
# refit=True: 최적의 파라미터를 찾은 후, 전체 데이터로 해당 모델을 재학습시킴
grid_search = GridSearchCV(model, param_grid, refit=True, verbose=2)

# 4. 학습 수행 (모든 조합을 시도)
grid_search.fit(X_train, y_train)

# 5. 결과 확인
print("최적의 파라미터: ", grid_search.best_params_)
print("최고 점수: ", grid_search.best_score_)

# 6. 최적의 모델 가져오기 및 예측
best_model = grid_search.best_estimator_
predictions = best_model.predict(X_test)
Method 02

랜덤 서치 (Random Search)

지정된 범위 내에서 무작위로 조합을 선택하여 성능을 테스트합니다. Bergstra와 Bengio의 논문(2012)에 따르면, 불필요한 탐색을 줄여 그리드 서치보다 효율적일 때가 많습니다. 특히 파라미터의 범위가 넓거나 연속적인 값을 가질 때 강력합니다.

Random search for hyper-parameter optimization (JMLR, 2012)

탐색 공간 시각화

범위 내 무작위 지점 탐색

random_search_example.py 
from sklearn.model_selection import RandomizedSearchCV  # 랜덤 서치 모듈
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier()

# 1. 탐색할 하이퍼파라미터 분포 설정
# 그리드 서치와 달리, 범위 내에서 랜덤하게 추출됩니다.
param_distributions = {
    'n_estimators': [10, 50, 100, 200],
    'max_features': ['auto', 'sqrt', 'log2'],
    'max_depth': [4, 5, 6, 7, 8],
    'criterion': ['gini', 'entropy']
}

# 2. RandomizedSearchCV 생성
# n_iter=10: 총 10번의 랜덤 조합만 시도하겠다는 의미 (속도 조절 가능)
random_search = RandomizedSearchCV(
    model, 
    param_distributions, 
    n_iter=10, 
    refit=True, 
    verbose=2
)

# 3. 학습 수행
random_search.fit(X_train, y_train)

# 4. 결과 출력 및 최적 모델 사용
print("최적의 파라미터: ", random_search.best_params_)
print("최고 점수: ", random_search.best_score_)

best_model = random_search.best_estimator_
predictions = best_model.predict(X_test)

Grid Search

모든 경우의 수를 다 확인하고 싶을 때 사용. 시간이 오래 걸리지만 꼼꼼함.

Random Search

탐색 범위가 넓거나 시간이 제한적일 때 사용. 효율적으로 최적값에 근접함.
