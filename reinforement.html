🧠정책 경사 알고리즘

Policy Gradient Update Rule

θt+1 := θt + α R(si) ∇θt log pθt(si)

강화학습에서 정책을 개선하는 핵심 알고리즘

📚각 기호의 의미

기호 의미
θt 현재 정책(Policy)의 파라미터 (즉, 신경망 가중치 등)
α 학습률 (learning rate)
R(si) 상태 si를 통해 얻은 보상(reward)
pθ(si) 파라미터 θ로부터 상태 또는 행동을 선택할 확률
∇θt θ에 대한 기울기(gradient) 연산
log pθ(si) 로그 확률 (계산을 안정화하고 곱셈을 덧셈으로 바꿈)

🎯핵심 아이디어

"좋은 결과(보상 R)를 준 행동은 더 자주 일어나게,
나쁜 결과를 준 행동은 덜 일어나게
정책 파라미터를 업데이트하자!"

✅ 보상이 높으면

해당 행동의 확률을 증가시킵니다

❌ 보상이 낮으면

해당 행동의 확률을 감소시킵니다

🧩예시로 풀어보기

가정: 동전 던지기 게임

📌 정책: "앞면이 나올 확률"을 조절하는 파라미터 θ
즉, pθ(앞면) = θ
🎁 보상 규칙:
• 앞면이 나오면 → R = +1
• 뒷면이 나오면 → R = -1
🎲 현재 상태: θt = 0.4 (앞면 나올 확률 40%)

🎯케이스 1: 앞면이 나왔을 때

1 시도: 동전을 던졌더니 앞면이 나왔다!
행동 si = 앞면, 보상 R(si) = +1
2 기울기 계산:
∇θt log pθt(si) = ∂/∂θt log(θt) = 1/θt
= 1/0.4 = 2.5
3 파라미터 업데이트: (α = 0.01 가정)
θt+1 = 0.4 + 0.01 × (1) × (2.5) = 0.425
결과: 앞면이 잘 나와서 보상이 좋았으니
앞면 확률이 40% → 42.5%로 증가! ⬆️

🔄케이스 2: 뒷면이 나왔을 때

1 시도: 동전을 던졌더니 뒷면이 나왔다
행동 si = 뒷면, 보상 R(si) = -1
2 기울기: 동일하게 2.5
3 파라미터 업데이트: (α = 0.01 가정)
θt+1 = 0.4 + 0.01 × (-1) × (2.5) = 0.375
결과: 뒷면이 나와서 보상이 나쁘니
앞면 확률이 40% → 37.5%로 감소! ⬇️

💬요약

항목 의미
R(si) 결과가 얼마나 좋았는가 (보상)
log pθ(si) 그 행동을 할 확률 (정책)
∇θ log pθ(si) 확률이 커지도록 정책 파라미터를 얼마나 바꿔야 하는가
전체 업데이트 보상이 좋으면 확률↑, 나쁘면 확률↓

"보상이 큰 행동은 더 자주 하게,
보상이 낮은 행동은 덜 하게"
